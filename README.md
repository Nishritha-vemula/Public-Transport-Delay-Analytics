# Public-Transport-Delay-Analytics

## Overview

This project analyzes public transport delay patterns using Power BI to identify trends, monitor operational performance, and generate actionable insights. The dashboard helps visualize delay behavior across different transport routes and supports data-driven decision making.

The objective of this project is to understand factors affecting delays and present insights through interactive visualizations.

---

## Dataset

The dataset contains information related to public transportation operations and delays.

### Features Included

- Route Information
- Transport Type
- Delay Duration
- Date and Time
- Location
- Traffic Conditions
- Peak Hours
- Delay Reasons

*(Dataset fields may vary depending on the source data.)*

---

## Tools Used

- Power BI
- Power Query
- DAX

---

## Project Steps

### 1. Data Import

- Loaded transportation data into Power BI

### 2. Data Cleaning & Transformation

Performed preprocessing using Power Query:

- Removed duplicates
- Handled missing values
- Changed data types
- Formatted columns
- Created calculated fields

### 3. Dashboard Development

Designed an interactive dashboard containing:

#### KPI Cards

- Total Delays
- Average Delay Duration
- Delay Percentage
- Route Performance Metrics

#### Visualizations

- Delay Distribution by Route
- Delays by Transport Type
- Delay Trends Over Time
- Peak Hour Analysis
- Location-wise Delay Analysis

#### Interactive Features

- Slicers for filtering
- Dynamic visual interactions
- Drill-down functionality

---

## Dashboard

The Power BI dashboard enables users to:

- Monitor transport delay trends
- Compare route performance
- Analyze delay patterns
- Identify high-delay areas
- Explore data interactively

---

## Key Insights

- Peak hours showed higher delay frequencies.
- Certain routes experienced more delays than others.
- Delay patterns varied across transport types.
- Interactive visualizations helped identify operational trends.
